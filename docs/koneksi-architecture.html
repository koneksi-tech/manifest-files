Koneksi platform architecture

Staging environment: GitOps-driven Kubernetes layout, application services, and backing data stores as defined in the manifest-files repository.

1. GitOps control plane

Declarative state lives in Git; Argo CD reconciles the cluster to match. Image updates can be automated via Argo CD Image Updater (staging tags on NHN Container Registry).

2. Kubernetes runtime (staging)

Application workloads run in koneksi-staging. Shared namespace and pull credentials are owned by the koneksi-backend Argo CD Application to avoid duplicate ownership warnings. Ingress exposes HTTP(S) via the cluster ingress controller.

5. Security & operations notes

  • Repository credentials and TLS material should not be committed; use argocd/SECRETS.md patterns, Sealed Secrets, or external secret stores.
  • Restrict LoadBalancers with cloud security groups and loadBalancerSourceRanges where supported.
  • Rotate any credentials that were ever exposed in Git history.
